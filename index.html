Frequently Asked Questions

var : Function-scoped, hoisted with undefined, re-declarable, mutable.
let: Block-scoped, hoisted but uninitialized, not re-declarable, mutable.
const: Block-scoped, hoisted but uninitialized, not re-declarable, immutable (value can't be reassigned).
map(): Returns a new array by applying a function to each element.
forEach(): Iterates over elements without returning a new array.
filter(): Returns a new array with elements that pass a given condition.
Syntax : Shorter and more readable (const add = (a, b) => a + b;).
this Binding : Lexically binds this (inherits from surrounding scope), unlike regular functions which have their own this.
No arguments Object : Arrow functions don’t have their own arguments, whereas regular functions do.
Implicit Return : Single-expression arrow functions return values without needing return.
A Promise in JavaScript represents a value that will be available in the future (asynchronous operation). It can be in one of three states:
Pending → Initial state, operation not completed.
Accept → Operation completed successfully.
Rejected → Operation failed.
A closure is when a function remembers the variables from its outer scope, even after the outer function has finished executing.
